初心者はどのエディタ or IDE を使ったらいいのか¶

はじめに¶

Clojure を書き始めようと思って最初に戸惑うのは開発環境でしょう。今までソフトウェア開発時に IDE を常用してきた人にとって、「 Leiningen と Emacs をインストールして、更に Cider-mode をインストールして .emacs.d/init.el を修正して、 cider-jack-in で Emacs 内に REPL を起動する…」ここまでしないと Clojure を書くことが出来ないというのは少々初心者にとってはハードルが高いどころか、そこまでしないといけないなら Clojure なんて書かないよ、と思われてしまうかもしれません。

そこで、改めて Clojure で何かを開発するときにどういった機能が必要か、もしくはどういう機能があると便利なのかを改めて整理し、それらを網羅出来てかつ初心者でも簡単に扱えるものというのを紹介したいというのがこの記事の主旨です。

エディタに必要な機能/あると便利な機能¶

先に列挙してしまいます。

必要な機能¶

  • シンタックスハイライト
  • 自動補完
  • nREPL へ接続出来る

あると便利な機能¶

  • キーバインドを変更出来る
  • 自動的に括弧の対応をとる
  • 強力な編集機能(構造的な編集)
  • 定義元ジャンプ
  • ドキュメントやソースを簡単に参照することが出来る
  • コードフォーマット
  • マクロ展開
  • リファクタリング
  • デバッガ

推敲中¶

勿論、今まで Emacs や Vim などを使っていてこれからも使いたい人はそれをそのまま使うのがいいと思います

ですが、 2015 年現在、今までエディタで開発をしてこなかった人に対して、 Emacs/Vim を薦めるのが本当にいいことなのかについては疑問が残ります。これまで Lisp 系言語と言えば Emacs を使った方がいいということはよく言われていました。またこれを書いている私自身もそう思います。しかし、 Emacs/Vim を使わなくても Clojure での開発をすることが出来る環境は整いつつあり、 Clojure コミュニティを見渡しても Emacs を使っている人は全体の半数程度と絶対的な立場を確立しているわけでもありません [1] 。

結論を言うと私は 2015 年現在であればテキストエディタを使いこなしているわけではないユーザーに対しては Cursive をお薦めしています。

ここからは何故数多あるテキストエディタ、あるいは IDE の中から Cursive をお薦めするのかというのを書いていきたいと思います。勿論、 Emacs や Vim という古くからあるテキストエディタを使うハッカーに憧れているという方たちは、テキストエディタを使った良いと思いますし、これより先を読む必要はないでしょう。

[1]日本ではかなりの人が Emacs を使っていますが、それは日本国内で Clojure ユーザーの絶対数が少ないためだと推測します。
